1. 对比下面两种写法。请说明：为什么第二种写成 `make_plan(goal, days)` 更方便复用、测试和修改？

   写法一：所有逻辑都写在主程序里。

   ```python
   goal = "练习 Python 函数"
   days = 3

   plan = []
   for day in range(1, days + 1):
       plan.append(f"第{day}天：学习 {goal}")

   for item in plan:

   ```

   写法二：把“生成计划”单独写成函数。

   ```python
   def make_plan(goal, days):
       plan = []
       for day in range(1, days + 1):
           plan.append(f"第{day}天：学习 {goal}")
       return plan


   plan = make_plan("练习 Python 函数", 3)
   for item in plan:
       print(item)
   ```

   ```python
   因为写法一要执行或者测试的话，会把整个文件都给执行一次，这样一会显的代码比较长，不方便查看，二是测试的时候需要每次都执行完整个文件，浪费资源。三是别人想用相同的逻辑时候，需要在写一份相同代码,造成冗余。使用写法二就不会有这种情况产生
   ```
