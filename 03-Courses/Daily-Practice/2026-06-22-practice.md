# 2026-06-22 每日练习

## 学习定位

- **为什么学这个**：PR2-01 是阶段 2「Prompt 工程」的第一课；L1 你已经会调 API 和看 token 了，现在要学怎么**写出好 prompt**——这是后面写摘要、做分类、锁 JSON 输出、以及一切 Agent 交互的底座。
- **学完能搞懂 / 能做什么**：能说出好 prompt 的 4 个要素（清晰指令 / few-shot 示例 / 角色扮演 / CoT），能判断一个 prompt 该不该加示例、加角色、加 CoT，以及什么时候写多了反而坏事。
- **对接下来的影响**：PR2-02 摘要 / PR2-03 分类 / PR2-04 JSON Schema 全是「拿这些技巧动手写 prompt 对比效果」——概念不扎实，后面写的 prompt 会凭感觉乱试。

## 生成依据

- 当前学习进度：L1-01~L1-05 全 PASS，PR2-01 概念预习（带读 4 块全 PASS）。
- 学习相关提问：带读中确认 few-shot 划清边界、指令调优让命令成立、角色扮演 vs 上下文示例、CoT 两个好处。
- 薄弱点 / RETRY：L1-05 曾把 temperature 的「差距压平」误述为「随机」(PARTIAL→PASS)；token 成本漏了缓存命中/未命中(RETRY→PASS)。本次概念预习暂无 RETRY。
- 关联旧知识点：`sampling-params-and-cost`（temperature 越高越发散）、`message-roles-and-instruction-hierarchy`（system prompt 角色）。
- 模型自拟迁移题：3 道——把 prompt 技巧迁移到具体场景（JSON 输出 / 数学题 / system prompt 与角色扮演的关系）。
