# LLM 本质与幻觉

## 一句话解释

大语言模型(LLM)本质就是一个"超级文字接龙玩家"：看着前面的内容，**预测下一个最可能出现的 token(词块)**。而"幻觉"(一本正经地胡说八道)就是这个机制**天生自带的副作用**，不是偶然的 bug。

## 地基：它只在干一件事

不管模型多大多复杂，每一步都只做一件事：

```text
看前文 -> 给所有候选词打概率 -> 挑一个"接上去最像样、最通顺"的词 -> 接回去，继续
```

抓住这条地基，第三章其它东西全是它的延伸：

- Transformer / 注意力机制 = 为了把"下一个词"猜得更准。
- Temperature / Top-p = 控制从候选词里"多稳 vs 多浪"地挑。
- 幻觉 = 这条地基的副作用(见下)。

## 为什么会幻觉（核心）

它挑的是「**最可能 / 最像样**」的词，但 **最可能 ≠ 最真实**。

- 它脑子里**没有"真假"这根弦**，只判断"像不像话"，不判断"是不是真的"。
- 它**没有"我不知道就闭嘴"的开关**，默认动作永远是接一个"听起来最顺"的答案。
- 所以哪怕**训练数据干净、你也没误导它**，遇到不知道的事，它照样会自信地编一个出来。

**直觉例子**：问它"我家那只猫叫什么名字？"——它不会说"我不知道"，多半会编一个像样的名字。因为"不知道"很少是"最像样的下一句"。

## 错误理解 -> 正确理解（易错点）

- ❌ 幻觉是模型"出错了 / bug" → ✅ 幻觉是"只求像、不求真"机制的**天生副作用**，无法靠"修 bug"根除，只能缓解。
- ❌ 只有数据有问题或被误导才会幻觉 → ✅ 数据干净、prompt 没问题时**照样**会幻觉。
- ❌ "上下文太长忘了前文"就是幻觉 → ✅ 那是**上下文窗口限制**(另一个问题)，会**加重**幻觉，但不是根因。详见 [[消息角色与指令优先级(Instruction Hierarchy)]]。

## 加重幻觉的诱因（不是根因，但会火上浇油）

- 训练数据里本身有错 / 互相矛盾(训练污染)。
- 误导性的 Prompt 把它带偏。
- 上下文太长，早期信息"顾头不顾尾"。

## 工程上怎么缓解

既然根因是"不求真、不会说不知道"，缓解思路都是**给它外部依据 / 约束**：

- **RAG**：先去检索真实资料，再让它基于资料作答(最常用)。
- 让它**调用外部工具**(计算器、搜索、数据库)拿确定结果。
- 多步推理 / 让它"先列依据再下结论"。
- Prompt 里明确允许它说"不确定 / 不知道"。
