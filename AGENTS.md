# AI-Knowledge 仓库约定

这是用户的长期 AI 知识库。用户要求“保存到知识库”“加入知识库”“整理进知识库”时，默认应保存到本仓库。

## 目录选择

- `00-Inbox`：未经整理的临时资料、网页摘录、对话摘要。
- `raw`：不可变原始资料层，保存网页、论文、字幕、源码摘录、对话摘要等 ingest 输入；整理后的知识再进入 `01-Maps` 到 `08-Memory`。
- `01-Maps`：总地图、主题地图、学习路线、索引。
- `02-Concepts`：一篇只讲一个概念的概念卡片，按主题域建二级目录，例如 `Python`、`LLM`、`Agent`、`MCP`、`RAG`。
- `03-Courses`：课程、教程、系列学习材料。先按知识点主题域建二级目录，例如 `Python`、`LLM`、`Agent`，再放课程或系列教程目录。
- `04-Projects`：实践项目、源码阅读、代码练习和实验记录。先按知识点主题域建二级目录，例如 `Python`、`LLM`、`Agent`，再放项目或源码阅读对象目录。
- `05-Papers`：论文阅读笔记。
- `06-Prompts`：提示词、工作流提示模板。
- `07-Reviews`：周复盘、月复盘、阶段总结。
- `08-Memory`：长期记忆候选、重要偏好、可复用结论。
- `Templates`：模板。

## Karpathy 式 LLM Wiki 工作流

本仓库优先按 Karpathy 式 LLM Knowledge Base / LLM Wiki 使用：

- `raw/` 是输入层，尽量保持原始资料不可变。
- `index.md` 是全局索引，给人和 LLM 快速定位入口。
- `log.md` 是操作日志，记录 ingest、结构调整、健康检查和重要维护。
- `AGENTS.md` 是规则文件，规定目录、链接、写作、同步和维护策略。
- `01-Maps` 到 `08-Memory` 是编译后的 wiki 层，由 LLM 协助维护、链接和扩写。

每次新增或整理资料时，遵循：

1. 原始资料先进入 `raw/` 或临时进入 `00-Inbox`。
2. LLM 根据资料更新已有 wiki 页，或新建概念卡、课程页、项目页、复盘页。
3. 新增核心页后更新 `index.md`、相关地图页或主题 README。
4. 完成后在 `log.md` 记录本次操作。
5. 对关键结论保留来源线索，避免只沉淀无来源的二手总结。

## 写作风格

- 使用 Obsidian Markdown。
- 优先加入 frontmatter：`type`、`topic`、`status`、`created`、`tags`。
- 重要笔记要补充相关链接，使用 Obsidian wikilink。
- 重要事实、课程结论、论文结论和工具结论应尽量补 `source` 或“来源/证据”小节。
- 如果新增核心资料，视情况更新 `01-Maps/AI 知识库总地图.md` 或相关主题地图。
- 新增概念卡时不要直接堆在 `02-Concepts` 根目录，应放入对应主题目录，并维护该目录下的 `README.md` 索引。
- 新增课程笔记时放入 `03-Courses/知识域/课程名/`；新增项目、代码练习、源码阅读时放入 `04-Projects/知识域/项目名/`。如果同一个学习执行区横跨多个知识域，应按具体笔记内容拆到对应知识域。

## 健康检查

每周或结构性更新后，运行一次知识库健康检查，并把结果放入 `07-Reviews/Knowledge-Base/`。检查重点：

- 断链、孤立页、空目录、缺 frontmatter。
- seed 笔记是否过多，哪些应扩写到 usable。
- 是否有重复概念或应该合并/拆分的笔记。
- 是否有关键结论缺来源。
- `index.md` 和相关地图是否仍然能代表当前结构。

## AI-Agent-Learning 同步规则

当检查 `C:\Users\26823\Desktop\AI-Agent-Learning` 的每日记录并判定任务为 `PASS` 时，必须同步稳定知识到本仓库：

- 新学会或被纠正的核心概念 -> `02-Concepts\<主题域>\`
- 课程路线、资料索引 -> `03-Courses\<主题域>\...`
- 代码练习、可运行产物、源码阅读记录 -> `04-Projects\<主题域>\AI-Agent-Learning\`

不要只写项目笔记或复盘而漏掉 `02-Concepts`。如果当天 PASS 任务涉及新的可复用概念，应新增或更新对应概念卡，并同步维护该主题目录下的 `README.md` 索引。

`03-Courses` 用来维护课程路线、阶段页和资料索引，不是每日作业归档。AI-Agent-Learning 任务 PASS 后，应按需更新对应课程页：

- 如果任务推进了学习阶段或小节状态，更新对应阶段页，例如 `03-Courses\Python\AI-Agent-Learning\stage0-python-basics.md` 的通过项、当前进度和下一步。
- 如果当天学习了新的课程资料、文档章节或视频系列，新增或更新对应课程索引/课程笔记，并链接到相关概念卡、项目产物或复盘。
- 如果当天只是完成代码练习，没有新的课程路线、资料索引或阶段状态变化，不要为了日期单独新建 `03-Courses` 笔记；应只更新已有阶段页或跳过 `03-Courses`。
- `03-Courses` 记录“学什么、路线到哪里、资料如何组织”，`04-Projects` 记录“做了什么代码/实验”，`07-Reviews` 记录“这次通过与问题复盘”。
