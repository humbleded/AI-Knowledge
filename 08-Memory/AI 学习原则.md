# AI 学习原则

## 背景

用户是一名软件开发工程师，正在学习 AI 和新技术。长期目标是成为 AI 开发工程师，同时提升工程能力，并保持持续学习，不追短期热点。

## 长期目标

- AI 开发工程师方向
- 提升工程能力
- 持续学习，而不是追热点

## 学习优先级

### 第一层：重点学习

- Python
- FastAPI
- Docker
- MCP
- AI Agent
- RAG
- 项目实战

### 第二层：工程基础支撑

- 数据库
- Linux
- 网络基础
- Git、CI、日志、测试、部署

这些内容要加入学习框架，但按项目需要逐步补齐，不替代 AI 开发主线。

### 第三层：保持关注

- OpenAI 更新
- Anthropic 更新
- Gemini 更新
- MCP 生态更新

### 第四层：暂不深入

- 刚发布的新框架
- 新 Agent 产品
- 新模型
- 短期热点

除非工作需要、项目需要或社区大量采用，否则只记录，不深入学习。

## 节奏原则

- 连续性比时长重要。
- 每天学习 10 分钟也算完成。
- 身体累就休息。
- 不追求每天学习很多。
- 避免因为压力过大而放弃学习。

## 新技术处理规则

当 AI 出现新技术时，先判断：

1. 它是什么。
2. 是否值得关注。
3. 是否应该进入学习队列。

不要默认把新技术加入学习计划。

## 回答偏好

后续学习规划、复盘、知识库整理和技术更新判断，都应优先遵守这套学习原则。
